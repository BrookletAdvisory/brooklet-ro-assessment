# Internal Supporting Note — Template

The internal note is the agent's evidence pack. It is for Brooklet's own file and for senior reviewer sign-off — **not** sent to the client.

## File format

- `.docx` in Brooklet Word style (Candara 12pt, A4, 1-inch margins).
- File naming: `[Applicant]-RO-Internal-Note.docx`.

## Language rules

- **Primary language: English.**
- **Bilingual entity names required** for all current and former employers, principals, and licensed corporations: format `English Name (中文名稱)` or `中文名稱 (English Name)`. This applies to the licence record table, CV ↔ SFC cross-check table, and the relevant-experience tables.
- CV phrasing in Chinese may be preserved verbatim where it materially affects the cross-check (e.g. job titles like 「副总经理」 may be retained alongside an English gloss).
- Numbers, dates, and SFC paper labels: English / Arabic.

## Acronym discipline

**On first occurrence in the document, write the full term followed by the acronym in parentheses.** Thereafter the acronym alone is fine.

| First occurrence example | Subsequent use |
|---|---|
| Regulated Activity (RA) | RA |
| Responsible Officer (RO) | RO |
| Licensed Representative (LR) | LR |
| Local Regulatory Framework Paper (LRP) | LRP |
| Recognised Industry Qualification (RIQ) | RIQ |
| Continuous Professional Training (CPT) | CPT |
| Hong Kong Permanent Resident (HK PR) | HK PR |
| Assets Under Management (AUM) | AUM |
| Politically Exposed Person (PEP) | PEP |
| Central Entity Number (CE No.) | CE No. |
| Managing Director (MD) | MD |
| Executive Director (ED) | ED |
| Non-Executive Director (NED) | NED |
| Fixed Income (FI) | FI |
| Asset Management Department (AM Dept) | AM Dept |

**Rule:** If you use a casual abbreviation (e.g. "Dy Head", "AM Dept") without first introducing the full term, that is a **defect**. Either expand on first use or write the full phrase throughout.

For an organisation name that already exists as a defined short form (e.g. BOCOM Securities = BOCOM International Securities Limited), introduce it once in a "Naming conventions used in this note" callout near the top of the document, then use the short form thereafter.

## Section structure (fixed order)

1. **Header table** — proposed principal, proposed RA(s) and role, candidates, sources retrieval date.
2. **Naming conventions** *(if any short forms are used for entities)*.
3. **Candidate 1 — [Name + 中文 + CE No.]**
   - 1.1 Overview
   - 1.2 SFC licence record
   - 1.3 CV / SFC cross-check
   - 1.4 Past-6-year relevant experience (separate table per RA)
   - 1.5 Management experience (≥ 2 years)
   - 1.6 Exam / exemption check *(one sentence — see exam-framework reference)*
   - 1.7 Practical issues
4. **Candidate 2 — [Name + 中文 + CE No.]** *(same sub-section structure as Candidate 1)*
5. **Comparative summary** *(only if 2+ candidates)*
6. **Pairing thesis** *(only if 2+ candidates)*
7. **Outstanding points / to be confirmed** *(per candidate)*
8. **Sources**

## Fixed table layouts

### Table 1 — SFC licence record

| Period | RA | Role | Principal |
|---|---|---|---|
| dd Mmm YYYY – dd Mmm YYYY | Type X | LR / RO | Entity short form (CE No.) |

Notes:
- Use the SFC Public Register's **"Date of licence"** as the licence start date for new licensees, and note the **"Date of accreditation to first principal"** separately if it pre-dates the formal licence date — both are important.
- Show ongoing entries as `Since dd Mmm YYYY`.
- Use entity short forms defined in the Naming conventions section.

### Table 2 — CV / SFC cross-check

| CV employer / period | SFC principal / period | Name check | Date / condition notes |
|---|---|---|---|

Apply these cross-check rules:
- Employer name match: identical / same group / different — flag if different.
- CV start vs licence start: if CV starts before licence, treat the pre-licence period as pre-licensing experience and note it.
- CV end vs licence end: if licence ends before CV end, take the licence end as the official end of the licence line.
- Continuous CV but broken licence: flag the gap.
- Conditions on SFC record: copy through and ensure the role description is consistent.

### Table 3 — Past-6-year relevant experience (one table per RA)

| Period | Substance | Relevance |
|---|---|---|

Use **Direct / Indirect / Partial / Not regulated-activity experience** as the relevance label.

After each RA's table, add a one-sentence italic conclusion:
> "*Type [X] view: direct experience approximately [N] years. [Above / below the 3-year benchmark]. [Satisfied / likely gap / TBC].*"

### Table 4 — Exam / exemption check

For most cases this is a **one-sentence paragraph**, not a table. Format:

> "Combined Type [X] + Type [Y] [LR/RO] requires N papers: Paper [list]. From [evidence], [the candidate] has demonstrably passed Paper [list] (or holds equivalent exemption). [Still needs to pass Paper Z] / [Within 3–8 yr re-entry window — no re-exam, only CPT top-up of N hours]."

If multiple sub-cases need separate treatment, fall back to a 3-column table: `Proposed RA | Current view | Paper / basis`.

## Outstanding points template

> **Candidate 1 Name:**
> 1. [Item — e.g. HKSI Paper 6 enrolment]
> 2. [Item — e.g. substance evidence pack for period X]
> 3. [Notice period / non-compete confirmation]
> 4. [Outside roles / shareholdings disclosure]
>
> **Candidate 2 Name:**
> 1. [CPT top-up confirmation — hours required]
> 2. [Project / AUM supplement — Brooklet to draft template]
> 3. [Co-RO substance arrangement]
> 4. [PEP enhanced DD pack]
> 5. [Reconciliation of any naming discrepancies]
> 6. [Target filing date]

## Sources section

List in this order:
1. Candidate CVs (note "on file" — do not paste content).
2. SFC Public Register links per candidate, with retrieval date.
3. Reference to SFC Guidelines on Competence (Oct 2024 version) and SFC Exam Requirements (the framework table).
4. Any other primary documents relied on.

## Quality checklist before delivery

Before sharing the file, confirm:

- [ ] All dates match the SFC Public Register exactly. Cross-checked Date of licence vs Date of first accreditation.
- [ ] No bare abbreviations introduced without first-occurrence expansion.
- [ ] Bilingual entity names provided for all relevant employers.
- [ ] Exam check section uses the canonical 6-paper / 4-paper framework — no invented combinations.
- [ ] Each candidate's section follows the 1.1–1.7 / 2.1–2.7 structure.
- [ ] Tables render cleanly at A4 with no severe column wrapping.
- [ ] No client-facing language has leaked into the internal note (or vice versa).
- [ ] No TBC fields silently filled with assumption.
