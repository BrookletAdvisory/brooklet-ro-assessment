# Client-Facing Preliminary Reply — Template

The client doc is a short, commercially presentable preliminary view. It is **not** an internal evidence pack.

## Unified structure (Chinese and English)

Both Chinese and English client docs follow the **same structure**:

1. **Salutation + opening line** — addressed to the client contact, stating that this is the preliminary RO assessment for the named applicant entity and the proposed RA(s).
2. **One paragraph per candidate** — short, commercial, covering suitability + headline strengths/gaps + headline exam position. No sub-bullets inside the candidate paragraphs.
3. **A closing block** combining:
   - a short **pairing thesis / overall recommendation** (one or two sentences); and
   - **other matters / 其他事项** — a short numbered list of practical follow-ups for the client.
4. **Sign-off line** — invites further discussion or document review.

The only differences between Chinese and English versions are language and idiomatic phrasing. The block layout is identical.

## Chinese template

```
[客户称呼]：

就贵司拟委任 [候选人 1 姓名] 与 [候选人 2 姓名] 担任 [申请实体名称] 项下 RA [X] + RA [Y] 负责人员（Responsible Officer，RO）一事，我们的初步评估意见如下：

[候选人 1 姓名]
[一段连贯的中文段落，约 4–8 句。涵盖：当前持牌状态 / 与拟申请 RA 的相关性 / 过去 6 年相关行业经验是否满足 / 管理经验是否满足 / 考试或豁免位置 / 1–2 项后续需关注的事项。语言条理化但不分项。]

[候选人 2 姓名]
[同上结构的另一段。]

总结及其他事项
[一两句配对/整体建议，例如：建议同时委任二人为 RO，互相补足。]
1. [实操建议 1]
2. [实操建议 2]
3. [客户需提供的下一步资料]

如以上理解有偏差或需要进一步核证，请告知。
```

## English template

```
Dear [Client Name],

With respect to the proposed appointment of [Candidate 1 Name] and [Candidate 2 Name] as Responsible Officers (RO) for [Applicant Entity] in connection with Type [X] + Type [Y] regulated activities, our preliminary assessment is as follows:

[Candidate 1 Name]
[One coherent English paragraph, approx. 4–8 sentences covering: current licensing status / fit to the proposed RA / past-6-year experience sufficiency / management experience / exam or exemption position / 1–2 follow-up items.]

[Candidate 2 Name]
[Same structure for the second candidate.]

Overall recommendation and other matters
[One or two sentences of pairing thesis / overall recommendation.]
1. [Practical recommendation 1]
2. [Practical recommendation 2]
3. [Items the client needs to provide]

We would be happy to discuss further upon your review.
```

## Style rules (apply to both languages)

- **First-occurrence acronym rule.** "Responsible Officer (RO)" or 「负责人员（Responsible Officer，RO）」 on first occurrence; thereafter the acronym alone is fine. Same convention for any other acronym used.
- **One paragraph per candidate.** No sub-bullets, no sub-headings, no internal flags inside the candidate paragraph.
- **Numbered list only in the closing block.** "总结及其他事项" / "Overall recommendation and other matters" is the only place a list is allowed.
- **No internal warnings or evidence trails.** Things like "TBC", "Brooklet to draft template", "evidence to substantiate" belong only in the internal note.
- **Tone: commercial, not regulatory-academic.** The client should feel they have received a clear preliminary view, not a regulatory essay.

## Length target

- Chinese: 600–900 字 total for a 2-candidate case.
- English: 400–600 words total for a 2-candidate case.

If significantly longer, internal-note material has likely overflowed. Trim back.

## What to put in the client doc — and what NOT to

| ✅ Include | ❌ Do NOT include |
|---|---|
| Bottom-line suitability conclusion per candidate | The full SFC licence record table |
| 1–2 sentence reason supporting suitability | The CV ↔ SFC cross-check table |
| Headline exam / exemption position (e.g. "still needs to pass Paper 6") | Detailed paper-by-paper analysis |
| Recency or licensing-gap headline (e.g. "off-register 17 months — within re-entry window") | CPT hour calculations and § references |
| Pairing thesis / overall recommendation | Internal flags such as "TBC", "evidence to substantiate", drafting notes |
| Practical follow-ups the client needs to action | Brooklet-internal action items (e.g. "Brooklet to draft template") |

## File format and naming

- Output as `.docx` in **Brooklet Word style** (Candara 12pt, A4, 1-inch margins, clean spacing). See the `brooklet-word-document-style` skill.
- File naming: `[Applicant]-RO初审回复-客户版.docx` (Chinese) or `[Applicant]-RO-Preliminary-Assessment-Client.docx` (English).
- One version only. Do **not** also produce a backup translation unless the user asks.
